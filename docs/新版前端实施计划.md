# 新版前端实施计划

本文档记录 `eSimulate-web` 的阶段性实施路线。当前目标是功能等价复刻旧版前端，同时使用现代 React 技术栈重建更合理的架构。

## 第一阶段：工程骨架

状态：已完成。

包含内容：

- Vite + React + TypeScript 项目配置。
- Ant Design 5、React Router 6、Axios、TanStack Query、Zustand、ECharts 依赖声明。
- Dockerfile、`.dockerignore`、Nginx 静态托管配置。
- `.env.example` 环境变量示例。
- 应用入口、Provider、主布局、登录布局。
- 第一阶段路由和侧边菜单。
- 本地启动脚本 `start-local.sh`。
- 登录、仿真、寻优、环境、模型、系统管理占位页面。
- 兼容旧后端路径的接口服务层。
- 认证 store、布局 store、API 类型、业务 DTO 类型。
- 路由菜单 smoke test。

验证命令：

```bash
npm run typecheck
npm test
npm run build
```

本地启动：

```bash
./start-local.sh
```

已知提醒：

- `npm install` 当前报告 5 个安全告警，暂未使用 `npm audit fix --force`，避免强制升级破坏依赖组合。
- 生产构建提示主 chunk 超过 500 kB。第一阶段可以接受，后续业务页面落地后通过路由懒加载和 manual chunks 优化。

## 第二阶段：基础业务模板

目标：

- 实现通用资源列表模板。
- 实现 CSV 上传、下载、删除和曲线预览。
- 实现模型配置页面模板。
- 实现通用图表组件。

第一批执行范围：

- 先实现环境/负荷资源配置表。
- 先实现电负荷页面作为样板页。
- 列表页先完成搜索、分页、选择、刷新、删除入口、上传入口、下载入口和曲线预览入口。
- 上传弹窗和曲线图先做通用组件外壳，后续接真实后端时再细化错误处理和图表数据适配。
- 测试优先覆盖资源配置、路由参数到配置的解析，以及电负荷页面是否使用旧版兼容 endpoint。

当前进展：

- 已建立 `environmentResourceDefinitions`，电负荷映射到旧版 `/load/electric-load-schemes` endpoint。
- 已建立通用 `ResourceListPage`，提供搜索、分页、选中、刷新、删除、下载、上传入口。
- 已建立通用 `CsvUploadModal`。
- 已建立通用 `CurvePreview`，使用 ECharts 并正确清理 resize 监听。
- `/environment/electric-load` 已切换为资源模板渲染。

优先页面：

- 电负荷。
- 热负荷。
- 风力数据。
- 水流数据。
- 光照数据。
- 温度数据。
- 风电模型。
- 光伏模型。

## 第三阶段：完整资源与模型复刻

目标：

- 补齐全部环境与负荷页面。
- 补齐全部模型配置页面。
- 对接旧版后端接口。
- 统一字段、枚举和 DTO 转换。

重点：

- 小水电公式计算。
- 电网电价。
- 火电机组。
- 热电联产。
- 抽水蓄能。
- 热储能。

## 第四阶段：方案仿真

目标：

- 复刻旧版方案仿真流程。
- 选择负荷、模型、环境。
- 配置模型数量。
- 调用 `/api/simulate/simulator/do`。
- 展示指标、饼图和堆叠图。

重点：

- DTO 组装逻辑必须做成纯函数并覆盖测试。
- 图表组件要避免旧版 resize 监听无法清理的问题。

## 第五阶段：寻优计算

目标：

- 复刻旧版 PSO 寻优流程。
- 创建任务、轮询状态、取消任务、获取结果。
- 展示寻优结果表格。

重点：

- 用 hook 管理任务生命周期。
- 组件卸载时停止轮询。
- 避免直接修改 React state。

## 第六阶段：系统管理与部署完善

目标：

- 根据业务实际需要复刻用户、角色管理。
- 完善 Docker/Nginx 部署方式。
- 增加 README、部署说明和更多测试。
